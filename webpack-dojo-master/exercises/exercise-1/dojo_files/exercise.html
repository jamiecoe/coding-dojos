Your First Tasks

So hopefully, you have a rough idea of what you're working with. To test your understanding, below you'll see a failing task in yellow. What's the issue? An explanation is provided immediately afterwards. If you know a bit of Webpack already, try making it pass before you read the solution. For debug output, you can click the "Fail" button. 


const path = require('path')

const inputPath = path.resolve(__dirname, 'src')
const outputPath = path.resolve(__dirname, 'dist')

module.exports = {
  context: inputPath,
  entry: {
    index: ['./index.js']
  },
  output: {
    path: outputPath,
    filename: '[name].js'
  }
}

We're doing a few additional steps than necessary to keep things organised. Here's the rundown.

Property Description
context Sets the root path to resolve all inputs.
entry A map of entry names to entry files. All the files are grouped into a single bundle.
output.path Where to write the final bundle.
output.filename The output filename for every entry bundle. We can use template strings here.

Entry Map

The entry map defines Webpack's starting points when it parses your source code. From these files, it'll read and gather all external dependencies (using require or import keywords).

The key of the map is the name used in the output file. The values are the source files to parse.

We'll go into this in detail in a later exercise. Right now, if you open the src directory, you'll see our source code has no dependencies.

The Output Path

The output.path isn't needed for this task as we're focusing on Webpack's output rather than any filesystem changes. This dojo will be entirely run in-memory so you won't see any files being written to disk. However, it's generally good practice to be explicit on where your files are going.

The Output Filename

The output.filename is also not needed for this task as [name].js is actually the default. I've added it to show how we can change the output names of the entry bundles.

Now for your next task. Read up on output.filename and change the output names so that it also contains a hash representation of the bundle such that the index entry becomes index-abc123.js.
